# 수업용 빙고 프린트 웹사이트 설계

작성일: 2026-07-18

## 목적

교사가 수업용 텍스트 항목을 입력하고, 학생용 빙고판을 여러 장 생성해 브라우저 인쇄로 출력할 수 있는 정적 웹 앱을 만든다. 앱은 서버 계정이나 원격 저장 없이 브라우저에서 동작하며, 같은 기기에서는 최근 입력값과 설정을 다시 불러올 수 있어야 한다.

## 대상 사용자와 핵심 흐름

대상 사용자는 교사다. 교사는 수업 주제에 맞는 단어, 문장, 숫자 등의 텍스트 항목을 줄 단위로 입력한다. 이후 빙고판 크기, 출력 장수, 무작위 배치 여부, 페이지당 배치 수, 용지 방향을 선택하고 학생용 빙고판을 생성한다. 마지막으로 브라우저 기본 인쇄 기능을 사용해 출력한다. 브라우저 인쇄 대화상자에서 종이 인쇄나 브라우저의 PDF 저장을 선택할 수 있지만, 앱이 PDF 파일을 직접 생성하지는 않는다.

핵심 흐름:

1. 제목과 항목을 입력한다.
2. `4x4` 또는 `5x5` 빙고판을 선택한다.
3. 같은 판 출력 또는 학생별 무작위 판 출력을 선택한다.
4. 생성할 장수를 직접 입력한다.
5. 페이지당 `1개`, `2개`, `4개` 배치와 `A4 세로`, `A4 가로` 방향을 선택한다.
6. 미리보기를 확인하고 브라우저 인쇄창을 연다.

## 범위

포함:

- SvelteKit/Vite 기반 정적 앱
- 텍스트 항목 입력
- `4x4`, `5x5` 빙고판 생성
- 같은 판 또는 무작위 판 출력 선택
- 출력 장수 직접 입력
- 페이지당 `1개`, `2개`, `4개` 배치 선택
- `A4 세로`, `A4 가로` 용지 방향 선택
- 제목과 이름칸 출력
- 브라우저 인쇄 지원
- `localStorage` 기반 브라우저 저장
- Dockerfile과 `docker-compose.yml` 기반 빌드 및 정적 서빙

제외:

- 로그인과 서버 저장
- PDF 파일 직접 생성
- 이미지 항목
- 정답지 또는 교사용 호출 목록 출력
- 5x5 가운데 `FREE` 칸

## 기술 방향

앱은 SvelteKit과 Vite로 작성하고 정적 빌드 산출물을 만든다. `@sveltejs/adapter-static`을 사용해 서버 API 없이 동작하는 정적 사이트로 빌드한다. Docker 배포는 멀티 스테이지 Dockerfile을 사용한다. 첫 단계에서 앱을 빌드하고, 두 번째 단계에서 Nginx 같은 경량 웹서버가 정적 파일을 서빙한다. `docker-compose.yml`은 로컬 또는 단일 서버 배포에서 컨테이너를 쉽게 실행할 수 있게 한다.

React는 사용하지 않는다. 사용자가 최종 지정한 프레임워크는 SvelteKit이며, SvelteKit은 Vite 기반으로 개발 서버와 빌드 파이프라인을 제공한다.

## 화면 구성

첫 화면이 곧 작업 화면이다. 별도 랜딩 페이지는 만들지 않는다.

데스크톱에서는 설정/입력 영역과 미리보기 영역을 나란히 배치한다. 모바일에서는 설정, 항목 입력, 미리보기가 세로로 이어진다. 교사용 반복 작업 도구이므로 장식적인 히어로 영역보다 입력 밀도, 검증 메시지, 인쇄 결과 확인을 우선한다.

주요 UI:

- 제목 입력 필드
- 빙고판 크기 선택: `4x4`, `5x5`
- 배치 방식 선택: 같은 판, 무작위 판
- 생성 장수 숫자 입력
- 페이지당 배치 선택: `1개`, `2개`, `4개`
- 용지 방향 선택: 세로, 가로
- 항목 입력 textarea
- 현재 항목 수와 필요한 최소 항목 수 표시
- 생성 버튼
- 인쇄 버튼
- 출력 미리보기

## 데이터 모델

설정 상태:

- `title`: 출력 제목
- `boardSize`: `4` 또는 `5`
- `randomizeBoards`: 학생별 무작위 판 여부
- `copyCount`: 생성 장수
- `boardsPerPage`: `1`, `2`, `4`
- `pageOrientation`: `portrait` 또는 `landscape`
- `itemsText`: 줄 단위 원본 입력값

생성된 보드:

- `id`: 화면 렌더링용 고유값
- `size`: `4` 또는 `5`
- `cells`: 빙고판 칸에 표시할 텍스트 배열

## 항목 처리와 생성 규칙

항목 입력은 textarea의 줄 단위 텍스트를 파싱한다. 빈 줄은 제거하고 각 줄의 앞뒤 공백은 정리한다. 중복 항목은 허용한다. 같은 단어를 여러 번 넣어야 하는 수업 상황이 있을 수 있기 때문이다.

필요 항목 수:

- `4x4`: 최소 16개
- `5x5`: 최소 25개

입력 항목이 필요 항목 수보다 적으면 생성과 인쇄를 막고 부족한 항목 수를 표시한다. 입력 항목이 필요 항목 수보다 많으면, 각 보드 생성 시 필요한 개수만 선택한다.

같은 판 모드에서는 한 개의 보드를 만든 뒤 요청한 장수만큼 동일하게 복제한다. 무작위 판 모드에서는 각 장마다 입력 항목 중 필요한 개수를 무작위로 선택하고 칸 순서를 섞는다.

5x5 보드는 가운데 무료 칸을 만들지 않는다. 모든 25칸에 교사가 제공한 항목이 들어간다.

## 인쇄 레이아웃

브라우저 인쇄를 기본 출력 방식으로 사용한다. `window.print()`로 인쇄창을 열고, `@media print` CSS에서 편집 UI를 숨기고 출력 영역만 보이게 한다.

인쇄 레이아웃은 선택값에 따라 CSS 클래스로 제어한다.

- 페이지당 1개: 한 장을 크게 출력
- 페이지당 2개: A4 한 페이지에 두 개의 빙고판 출력
- 페이지당 4개: A4 한 페이지에 네 개의 빙고판 출력
- 세로/가로: `@page`의 `size`와 출력 그리드 비율 조정

각 빙고판에는 제목과 이름칸이 포함된다. 이름칸은 학생이 직접 이름을 쓸 수 있도록 빈 선 또는 빈 영역으로 제공한다.

## 브라우저 저장

설정과 항목 입력값은 변경될 때 `localStorage`에 저장한다. 앱을 다시 열면 저장된 값을 복원한다. 저장 대상은 생성된 보드 결과가 아니라 교사의 입력값과 설정값이다. 이렇게 하면 다시 열었을 때 최신 설정으로 새 보드를 생성할 수 있다.

브라우저가 `localStorage`를 차단하거나 private mode에서 저장에 실패해도 앱 사용은 계속 가능해야 한다. 이 경우 저장만 생략하고, 설정 영역 근처에 자동 저장을 사용할 수 없다는 짧은 안내를 표시한다.

## 예외 처리

검증 오류는 사용자가 바로 고칠 수 있는 문구로 표시한다.

- 항목 부족: `4x4 빙고판은 항목이 16개 이상 필요합니다.` 또는 `5x5 빙고판은 항목이 25개 이상 필요합니다.`
- 생성 장수 오류: `1장 이상 입력해 주세요.`
- 생성 장수 상한 초과: `최대 100장까지 생성할 수 있습니다.`
- 저장 실패: `이 브라우저에서는 자동 저장을 사용할 수 없습니다.`

생성 장수는 1장 이상 100장 이하로 제한한다. 너무 많은 DOM과 인쇄 페이지를 한 번에 만들면 브라우저가 느려질 수 있기 때문이다.

## 컴포넌트 구조

- `SettingsPanel`: 제목, 판 크기, 무작위 여부, 출력 수, 페이지당 배치 수, 용지 방향 설정
- `ItemsEditor`: 줄 단위 항목 입력과 항목 수 표시
- `PreviewToolbar`: 생성, 인쇄 버튼과 검증 메시지 표시
- `BingoBoard`: 단일 빙고판 렌더링
- `PrintPreview`: 여러 보드를 페이지당 배치 규칙에 맞춰 렌더링
- `utils/bingo`: 항목 파싱, 무작위 선택, 셔플, 보드 생성
- `stores/settings`: `localStorage` 저장과 복원

## 테스트 전략

우선 로직 테스트를 중심으로 작성한다.

- 항목 파싱: 빈 줄 제거와 공백 정리
- 최소 항목 수 검증: `4x4`는 16개, `5x5`는 25개
- 초과 항목 처리: 필요한 개수만 선택
- 같은 판 모드: 모든 출력 보드가 같은 셀 순서를 가짐
- 무작위 판 모드: 각 보드가 독립적으로 생성됨
- 출력 장수 검증: 1장 미만과 100장 초과는 생성 불가

구현 후 검증:

- `npm run build`
- 로컬 앱 실행
- 데스크톱과 모바일 화면 확인
- 브라우저 인쇄 화면에서 편집 UI가 숨겨지고 빙고판만 출력되는지 확인

## 성공 기준

- 교사가 텍스트 항목을 입력해 `4x4` 또는 `5x5` 빙고판을 생성할 수 있다.
- 항목이 최소 개수보다 적으면 생성 전에 명확한 오류가 표시된다.
- 같은 판 또는 학생별 무작위 판을 선택할 수 있다.
- 출력 장수를 직접 입력할 수 있고 최대 100장까지 생성된다.
- A4 세로/가로와 페이지당 1/2/4개 배치가 인쇄 미리보기에 반영된다.
- 브라우저 인쇄 시 편집 UI 없이 빙고판만 출력된다.
- 최근 입력값과 설정이 같은 브라우저에서 복원된다.
- Docker Compose로 빌드된 정적 앱을 실행할 수 있다.
